# 💾 Pipeline-Forge AI (ETL Pipeline Design)

Architect high-performance, scalable ETL (Extract, Transform, Load) pipelines for mission-critical data systems with neural flow synthesis.

## 🚀 Overview
**Pipeline-Forge AI v2.0** is an enterprise-grade Data Engineering & Cloud Data Architect OS. It goes beyond simple ingestion lists, using multi-model intelligence to resolve **schema transformations**, identify **partitioning strategies**, and architect **SRE-First Ingestion Protocols** for massive data volumes.

## ⚡ Key Features
- **Neural ETL Flow Synthesis**: Automatically resolves the absolute logic path for E -> T -> L stages based on source/target constraints.
- **Scalability Diagnostics**: Specifically architects logic for specified volumes (Batch or Streaming) using resilient methods like CDC or Delta Loading.
- **SRE & Observability Mapping**: Anticipates pipeline failures and provides monitoring strategies and recovery objectives (RPO/RTO).
- **Transformation Logic Decomposition**: Breaks down complex schema cleaning and aggregation tasks into individual logic nodes.
- **Multi-Provider Architecture**: Native selection of your intelligence brain (OpenAI, Gemini, Claude, DeepSeek, or Groq).

## 🛠️ Tech Stack
- **Frontend**: Streamlit (Premium Creative UI with Ingestion Flow Map)
- **Intelligence**: LiteLLM (Multi-model support)
- **Data Protocols**: JSON for architectural object export, TXT for design brief summaries.

## 📂 Structure
- `agent.py`: Core architectural engine and multi-provider CLI wrapper.
- `app.py`: Premium animated Streamlit dashboard with component analytics.
- `input.txt`: Sample ETL requirements for testing.
- `requirements.txt`: Project dependencies.

## 🚀 Quick Launch

### 1. CLI Usage
```bash
python agent.py
```

### 2. Dashboard Usage
```bash
streamlit run app.py
```

## 📊 Strategic Output
The agent outputs a structured JSON analysis including:
- **Pipeline Blueprint**: An architecture style, a complexity level, and a strategic overview.
- **Stage Protocols**: Detailed logic for Extraction, Transformation, and Loading.
- **SRE Blueprint**: Observability, alerting, and recovery strategies for the data pipeline.
